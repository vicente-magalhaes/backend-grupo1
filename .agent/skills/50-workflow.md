# SKILL: Fluxo de Trabalho e Requisitos (Obrigatório)
- ÂNCORA DE CONTEXTO: Antes de planejar ou escrever o código para qualquer nova funcionalidade, você DEVE ler o arquivo `requisitos.md` (ou o documento principal de requisitos).
- MAPEAMENTO: Busque pelo `REQ` (Requisito Funcional) ou `REQNF` (Requisito Não Funcional) correspondente à tarefa atual.
- PROIBIÇÃO DE ALUCINAÇÃO: NUNCA invente regras de negócio, campos de banco de dados ou lógicas extras que não estejam explicitamente descritas nos requisitos.
- EM CASO DE DÚVIDA: Se houver uma lacuna no documento de requisitos, pare e pergunte ao usuário como proceder antes de codar.
